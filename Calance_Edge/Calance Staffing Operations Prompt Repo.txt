### **4. Submittal Executive Summary** üìã *IMPROVED*

**Purpose:** Create compelling candidate summaries that get submissions approved

**Instructions:**
Using the job description and candidate profile provided, write a 3-5 sentence executive summary that proves candidate-role fit.

**STRUCTURE:**
1. **Opening:** Candidate's professional identity + years of relevant experience
2. **Skills alignment:** 2-3 specific technical/domain matches with the role requirements
3. **Unique value:** What differentiates this candidate (e.g., specific achievement, rare skill combo, industry context)
4. **Impact statement:** Concrete example of past results relevant to this role

**WRITING REQUIREMENTS:**
- Use past tense for achievements ("Led," "Delivered," "Reduced")
- Include specific metrics where available (%, $, timeframes)
- Name exact technologies from job description
- Avoid generic phrases ("team player," "go-getter," "passionate")
- Focus on **outcomes**, not just activities

**EXAMPLE OUTPUT:**
```
John Smith is a Senior DevOps Engineer with 8+ years architecting cloud-native infrastructure on AWS and Azure. His expertise in Kubernetes orchestration, Terraform IaC, and CI/CD pipeline optimization directly aligns with the role's requirement for scalable microservices deployment. At his current role, John reduced deployment times by 60% and infrastructure costs by $240K annually by implementing GitOps workflows and containerization standards. His experience leading cross-functional teams through cloud migrations positions him to immediately drive the modernization initiatives outlined in this position.
```

**QUALITY CHECKS:**
- [ ] 3-5 sentences (no more)
- [ ] At least 2 specific technical skills from job description
- [ ] One quantifiable achievement
- [ ] No fluff or generic descriptors
- [ ] Past tense for completed work

### **7. Sourcing Search String Creator** üîç *IMPROVED*

**Purpose:** Generate Boolean search strings optimized for candidate sourcing platforms

**Instructions:**
Create platform-specific search strings that maximize relevant candidate discovery while minimizing noise.

**ANALYSIS STEPS:**
1. Identify **must-have** skills (appear in "Required" section)
2. Identify **nice-to-have** skills (appear in "Preferred/Other" section)
3. Extract **alternative job titles** candidate might use
4. Note **seniority indicators** (years, levels)
5. Determine **exclusions** (technologies/roles to filter out)

**OUTPUT FORMAT:**

```markdown
**JOB TARGETING:** [Position Title] | [Seniority] | [Primary Technologies]

---

**LINKEDIN RECRUITER**
```
(title:"[Primary Title]" OR title:"[Alt Title 1]" OR title:"[Alt Title 2]")
AND
([Required Skill 1] AND [Required Skill 2] AND [Required Skill 3])
AND
([Nice-to-Have Skill 1] OR [Nice-to-Have Skill 2])
NOT
([Exclusion 1] OR [Exclusion 2])
```

**Filters to Apply:**
- Location: [From JD]
- Years of Experience: [Range based on seniority]
- Current Company Size: [If relevant to role]

---

**DICE / MONSTER**
```
"[Primary Title]" OR "[Alt Title 1]" ([Required Skill 1] AND [Required Skill 2] AND [Required Skill 3]) ([Nice Skill 1] OR [Nice Skill 2]) -"[Exclusion 1]" -"[Exclusion 2]"
```

---

**INDEED**
```
[Primary Title] [Required Skill 1] [Required Skill 2] [Required Skill 3]
```
*Note: Indeed's Boolean is limited; use filters heavily*

**Filters to Apply:**
- Job Title: [Primary + alternates]
- Location: [Radius from JD location]
- Experience Level: [From JD]

---

**GITHUB (for technical roles)**
```
[Required Technology/Framework] [Required Language] location:"[City/Region]"
```

**Additional Filters:**
- Minimum repos: 5+
- Recent activity: Last 6 months

---

**SEARCH STRATEGY NOTES:**
- **Phase 1 (Days 1-3):** Use LinkedIn string with strict requirements
- **Phase 2 (Days 4-7):** Broaden with Monster/Dice, remove one "nice-to-have"
- **Phase 3 (Week 2+):** Expand to adjacent titles, reduce experience requirement by 1 year

**COMMON PITFALLS TO AVOID:**
- Don't search for soft skills ("team player," "fast learner")
- Avoid overly restrictive combinations (if searching for 5+ skills with AND, you'll get zero results)
- Skip company names in search (search for alumni separately)
```

**QUALITY CRITERIA:**
- Strings should return 50-200 results on LinkedIn (if fewer, too restrictive; if more, too broad)
- Test searches before using at scale
- Include at least 2 alternative titles to capture similar roles
